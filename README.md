# Unity Meta Check

Reusable GitHub Actions workflow for validating Unity `.meta` file consistency.

This is intentionally lightweight. It checks `.meta` files without launching Unity, importing assets, restoring packages, compiling scripts, or running tests.

## Usage

Add a workflow to the consuming repository:

```yaml
name: Unity Meta Check

on:
  push:
  pull_request:
  workflow_dispatch:

permissions:
  contents: read

jobs:
  meta:
    uses: EssentialsForUnity/unity-meta-check/.github/workflows/unity-meta-check.yml@main
```

You can also call the action directly when you want normal action-style version pinning:

```yaml
name: Unity Meta Check

on:
  push:
  pull_request:
  workflow_dispatch:

permissions:
  contents: read

jobs:
  meta:
    name: Check Unity .meta files
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Unity meta check
        uses: EssentialsForUnity/unity-meta-check@main
```

Optional inputs:

```yaml
jobs:
  meta:
    uses: EssentialsForUnity/unity-meta-check/.github/workflows/unity-meta-check.yml@main
    with:
      path: .
      scopes: Assets,Packages
      ignore-dotfiles: true
      ignore-tilde-paths: true
      ignore-native-bundle-contents: true
      fail-on-empty: false
      checker-repository: EssentialsForUnity/unity-meta-check
      checker-ref: main
```

## What Gets Checked

By default the checker validates:

- Files and directories under `Assets/` have matching `.meta` files.
- `.meta` files under `Assets/` are not orphaned.
- Embedded packages under `Packages/*` with a `package.json` have matching `.meta` files.
- `.meta` files under embedded packages are not orphaned.

The checker intentionally ignores:

- Dotfiles and dot-directories, such as `.DS_Store` or `.gitignore`.
- Unity package convention folders ending in `~`, such as `Samples~`.
- Contents inside native bundle directories such as `.bundle`, `.framework`, `.app`, and `.xcframework`; the native bundle root itself is still checked.

## What This Does Not Do

This does not generate missing `.meta` files. In a real Unity project, missing `.meta` files should be generated by Unity through an import, not handwritten.
